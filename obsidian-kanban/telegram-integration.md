# Telegram — интеграция

**Статус:** принято (2025-06-25)  
**Связано:** `project-dev.md` → Backlog → Telegram

## Решение

Интеграция Telegram строится на **Bot API**: пользователь создаёт бота через [@BotFather](https://t.me/BotFather), передаёт **токен бота** в приложение и **добавляет бота администратором** в канал.

**MTProto** (`api_id` / `api_hash`, авторизация по телефону) **отложен** — пока нет возможности получить и использовать эти credentials. К MTProto вернёмся, когда ситуация разрешится.

## Что даёт Bot API (наш scope)

| Возможность | Bot API | Статус в продукте |
|-------------|---------|-------------------|
| Число подписчиков канала | `getChatMembersCount` (бот — админ) | **Цель фазы 1** — Live в блоке подписчиков |
| Проверка доступа бота к каналу | `getChat`, `getChatMember` | **Цель фазы 1** |
| Просмотры / реакции / forwards поста | Нет в Bot API | **Ручной ввод** до появления MTProto |
| Агрегированная аналитика канала | Только MTProto (`stats.*`) | **Отложено** |

## UX для пользователя (фаза 1)

1. **Подключить бота** (один раз): настройки → токен от BotFather → проверка `getMe`.
2. **Добавить канал**: ссылка `t.me/…` или `@handle` → чеклист «добавьте бота админом» → кнопка «Проверить».
3. **Подписчики**: Live-синхронизация при открытии дашборда (как YouTube).
4. **Публикации**: ссылка на пост `t.me/channel/123`, метрики — **вручную** (views → просмотры, реакции → likes).

## Архитектура (план реализации)

- Новая сущность `TelegramBotConnection` (аналог `OAuthConnection`): `botTokenEnc`, `botId`, `botUsername`, `status`.
- `SubscriberSource` для `TELEGRAM` ссылается на подключение бота; `trackingMode: AUTOMATIC` после успешной верификации.
- `TelegramModule` / `TelegramService` — HTTP к `api.telegram.org/bot<token>/…`.
- В `shared`: правило авторизации `bot_token` (после реализации заменит `manual_only` для подписчиков).
- Метрики публикаций: `PROVIDER_METRIC_AUTH[TELEGRAM]` остаётся `manual_only` до MTProto.

## Отложено (MTProto)

- Автоматические просмотры и реакции постов (`stats.getMessageStats`).
- Сводная статистика канала (`stats.getBroadcastStats`).
- Любой flow с телефоном / `api_id` / `api_hash`.

## Ограничения

- Бот должен быть **админом** каждого отслеживаемого канала.
- Один токен бота на пользователя — несколько каналов, если бот админ везде.
- Токен хранится только на бэкенде, зашифрован (как OAuth-токены).
- Приватные каналы без `@username` — через numeric `chat_id` после добавления бота.
- **Прокси опционален:** `TELEGRAM_API_PROXY` в `apps/api/.env` — только если API-сервер не достучится до `api.telegram.org` (локальная разработка). На зарубежном VPS переменную не задают. Поддерживаются `http://`, `https://`, `socks5://`, `socks5h://`.
